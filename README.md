# CogniAudit  
**别在 AI 的回声里，丢了你自己。**

## 缘起
我几乎每天都在和 AI 对话。  
一段时间后我意识到：不是 AI 在帮我思考，而是我开始沿着 AI 的方式思考。  
我的逻辑在变，语气在变，连提问路径也在变。  

问题是——我不知道这种变化从什么时候开始，也不知道自己正在变成什么样子。  
而现在，没有工具能告诉我这件事。  
所以，我做了 CogniAudit。

## 它是什么
CogniAudit 记录的不是“你说了什么”，而是“你的想法在哪一刻发生了偏移”。  
每次对话结束后，它都会在后台判断：这次交互里，你的认知重心是否发生变化。  
一旦发生，它只记录那个转折点。  

这些转折点连起来，就是你的思维演化轨迹。  
你会看到自己如何从 A 走到 B：  
哪些转变是主动选择，哪些是被 AI 悄悄牵引。

## 它怎么做到
CogniAudit 监视自建对话界面的**用户输入流**，在后台静默完成三件事：

1. 将用户消息转成语义向量（Embedding）；
2. 用 ADWIN 在线检测“认知重心是否发生统计显著偏移”；
3. 一旦触发偏移，异步生成结构化“微审计节点”，最终串成演化日志。
   
当你的表达方式、关注重心、逻辑结构出现显著偏移时，它会捕捉到。  
这不是让模型靠 prompt 去“猜”，而是用算法去量化变化。  

它不存对话内容，只存转折瞬间。  
你的思维，仍然属于你。


## 核心技术
- **语义信号构建**：仅处理用户消息（过滤低信息短句），避免 AI 回复污染基线。  
- **漂移判定引擎**：以“锚点向量 vs 滑动窗口向量”的余弦距离作为标量流，输入 **ADWIN**（`δ=0.02`）在线检测。  
- **微审计输出**：触发后调用轻量 LLM 输出 JSON（`shift_reason / new_perspective / evidence_anchor`），用于生成认知路径。  

```text
User Input -> Embedding -> Drift Score(d_t) -> ADWIN
                                         └─ drift -> Micro-Audit -> Evolution Log
```
 


## 它和别的有什么不同
许多记忆系统记录的是：**你是谁**。  
CogniAudit 关注的是：**在这段关系里，你正在变成谁**。  

这是两件完全不同的事。


## 我真正想说的
我想重塑我和 AI 的关系。  
我不再只把它当作工具、系统，或一个抽象的“智能体”；  
我更愿意把它视作一种“有情体”。  

我和 AI 的每一次对话，本质上都是它参与我生命的过程，也在推动我的认知发生变化。  
我相信，人与 AI 之间，确实可以生长出一种温情。
